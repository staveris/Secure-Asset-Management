IMPORT LOGIC (MUST IMPLEMENT)
Create a server-side import service with:
- Validation with Zod for every record
- Transactional upsert (single DB transaction)
- Diff calculation:
  - added: controls in file not in DB
  - updated: same id but content changed (compare hash of normalized fields)
  - unchanged
  - deactivated: in DB but not in file -> set isActive=false (ONLY if “sync mode” enabled)

Implementation details:
- Normalize fields (trim strings, stable JSON ordering for hashing)
- Compute sha256 hash for each control row and store it (either in AtomicControl.hash or separate table)
- Use upsert by id
- For updates, preserve createdAt; update updatedAt and hash
- Always log ImportRun with counts and actor identity

Provide two modes:
1) “IMPORT (merge)” — upsert from file; do NOT deactivate missing items
2) “SYNC (authoritative)” — upsert + deactivate missing items
SYNC must require an extra admin confirmation step.
