OPTION B CONTROL EXTRACTION (ADD-ON GENERATOR)
Add scripts (do NOT run automatically on every boot):
- /scripts/atomic/generate_nis2_option_b.ts
- /scripts/atomic/generate_cir_2024_2690_option_b.ts
- /scripts/atomic/run_all.ts

Authoritative sources:
- NIS2 2022/2555 PDF: https://eur-lex.europa.eu/legal-content/EN/TXT/PDF/?uri=CELEX:32022L2555
- CIR 2024/2690 PDF: https://eur-lex.europa.eu/legal-content/EN/TXT/PDF/?uri=OJ:L_202402690

Implementation rules:
- Parse PDF text (pdf-parse or similar)
- Normalize whitespace
- Split into: Article → paragraph → list item (a-j) → nested (i-iv)
- Create one AtomicControl per atomic clause

Deterministic ID rules:
- NIS2: "NIS2-<article>-<para>[-<letter>][-<roman>]"
  e.g. NIS2-20-1, NIS2-21-2-c, NIS2-23-4-d-iii
- CIR Annex: "CIR-2024-2690-ANNEX-<section>-<sub>-<item>" (deterministic)
Store legalRef and clausePath for traceability.
obligationText must be a short paraphrase or a trimmed snippet <= 300 chars.

Applicability tagging:
- NIS2 controls: tags include "ALL_NIS2_ENTITIES"
- CIR controls: tags include applicable entity types (DNS/TLD/CLOUD/DC/CDN/MSP/MSSP/ONLINE_PLATFORM/TRUST)
- Add applicability JSON rules that match your Tenant.profile flags and sector/subsector taxonomy.

EvidenceTypes & testProcedure defaults:
- Provide reasonable defaults based on article/family.
All fields must be editable later by PLATFORM_ADMIN.
