Applicability logic:
Implement a deterministic DORA applicability engine.

Core rule:
DORA controls must not appear in an organization’s active assessment unless dora_enabled = true.

Set dora_enabled = true only when:
1. dora_scope_confirmed = true, and
2. dora_article2_in_scope = true or ict_third_party_provider_profile = true, and
3. dora_article2_exclusion = false.

If dora_article2_exclusion = true:
- keep dora_enabled = false
- show the organization as “DORA not applicable / excluded”
- do not create active DORA assessment controls
- do not remove or affect any NIS2/NIST controls

If dora_article16_simplified = true:
- assign controls tagged DORA_SIMPLIFIED
- do not assign controls tagged only DORA_FULL
- still allow additional relevant controls based on other tags where applicable, such as THIRD_PARTY, ROI, PAYMENT_RELATED_ENTITY, OPTIONAL, ICT_TPP, or CTPP, only if their corresponding applicability flags are true

If dora_article16_simplified = false and dora_enabled = true:
- assign controls tagged DORA_FULL

Additional tag rules:
- Assign TLPT_SELECTED controls only when tlpt_selected_or_required = true
- Assign THIRD_PARTY controls only when uses_ict_third_party_services = true
- Assign ROI / Register of Information controls when uses_ict_third_party_services = true
- Assign CRITICAL_IMPORTANT_FUNCTION controls only when has_critical_or_important_functions = true or ict_services_support_critical_or_important_functions = true
- Assign PAYMENT_RELATED_ENTITY controls only when payment_related_entity = true
- Assign OPTIONAL information-sharing controls only when participates_in_information_sharing = true
- Assign ICT_TPP controls only when ict_third_party_provider_profile = true
- Assign CTPP controls only when critical_ict_third_party_provider_designated = true

Do not treat every NIS2 entity as DORA applicable.
Do not treat every organization in the financial sector as automatically DORA applicable unless scope has been confirmed.
Provide an admin override, but require a reason/note when manually enabling or disabling DORA.
